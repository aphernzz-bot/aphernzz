Módulo 1 — Fundamentos: qué automatizar y por qué

Antes de automatizar nada, necesitas saber qué vale la pena automatizar. Este módulo te da el mapa para no desperdiciar tiempo en las tareas equivocadas.

El ROI de la automatización: cuánto tiempo estás perdiendo

Calculamos juntos cuántas horas semanales se van en tareas repetitivas y las convertimos a costo real. Una hoja de cálculo simple que muchos directores nunca han hecho.

Criterios para elegir: ¿Make, Python o los dos?

Regla práctica: si puedes hacerlo con clics en Make, úsalo. Si necesitas lógica compleja, transformar datos o conectar una API sin módulo oficial, usa Python. Verás ejemplos de cada caso.

Módulo 2 — Make desde cero

Make (antes Integromat) es una de las herramientas más potentes para automatización visual. Puedes construir flujos complejos sin escribir una sola línea de código. Aquí aprendes a usarla de verdad.

Crear tu cuenta y entender la interfaz de Make

Configuras tu cuenta gratuita de Make, entiendes qué son los escenarios, los módulos, los disparadores y las operaciones. La interfaz visual al principio puede confundir, pero en 20 minutos ya te mueves con confianza.

Video — 20 min

Primer escenario: formulario web → Google Sheets

Cada vez que alguien llena un formulario de contacto en tu web, los datos aparecen automáticamente en una hoja de cálculo con nombre, correo, teléfono y fecha. Sin copiar y pegar.

Segundo escenario: notificación de pago por email y WhatsApp

Cuando tu sistema de cobro (Stripe, MercadoPago, o una hoja de cálculo manual) registra un pago, el cliente recibe un correo de confirmación y tu equipo recibe un mensaje en WhatsApp o Slack.

Escenario funcional

Webhooks: cómo recibir datos de cualquier sistema

Aprendes a crear un webhook en Make para recibir datos de apps que no tienen módulo oficial. Si la app puede hacer una solicitud HTTP, puede conectarse a Make. Esto lo amplía todo.

Módulo 3 — Make avanzado: flujos condicionales y errores

El siguiente nivel de Make está en sus routers e iteradores. Con estos dos conceptos puedes construir automatizaciones que reemplazan procesos completos de un departamento.

Routers: distintas acciones según la condición

Un mismo escenario puede enviar un lead a diferentes vendedores según el estado del cliente, asignar una etiqueta diferente según el monto de la compra o notificar a distintos canales según la urgencia.

Video — 18 minEscenario de ejemplo

Iteradores y agregadores: procesar listas de datos

Tienes 50 pedidos nuevos en un sheet y quieres procesar cada uno individualmente. Los iteradores dividen la lista, los agregadores la vuelven a juntar. Poderoso para reportes y sincronizaciones masivas.

Video — 22 min

Manejo de errores: que nada falle en silencio

Configuras directivas de error para que Make te notifique cuando algo sale mal, reintente automáticamente o continúe con el siguiente elemento sin detener todo el escenario.

Video — 15 min

Escenario completo: sincronización CRM y reporte diario

Un escenario real que cada mañana revisa los leads del día anterior en tu CRM, calcula cuántos están en cada etapa del pipeline y envía un resumen por correo a los directores.

Módulo 4 — Python para automatización

No vas a aprender Python completo. Vas a aprender las partes de Python que sirven directamente para automatizar: leer archivos, llamar APIs, manipular datos y programar tareas. Nada más.

Python para quienes nunca han programado

Variables, strings, listas, condicionales y funciones. Todo explicado con ejemplos de negocio: no "imprimir hola mundo" sino "leer el nombre del cliente de un archivo y armar un saludo personalizado".

Video — 25 minEjercicios guiados

Automatizar Excel con openpyxl: leer, escribir y formatear

Lees un Excel de ventas, calculas totales por vendedor, agregás una columna de comisión calculada y guardas el resultado en un nuevo archivo. Un script que se ejecuta con doble clic.

Enviar correos automáticos con Python

Con smtplib y email.mime envías correos personalizados con adjuntos desde Python. Ejemplo práctico: envío masivo de facturas individuales a una lista de clientes en un Excel.

Módulo 5 — Python + APIs

Una API es una puerta de acceso a los datos de otra aplicación. Una vez que sabes cómo funcionan, puedes conectar casi cualquier herramienta con cualquier otra. Este módulo te abre esa puerta.

Qué es una API y cómo leer su documentación

Desmitificamos las APIs con un lenguaje humano. Endpoint, método HTTP, autenticación, parámetros, respuesta JSON. Lo suficiente para que puedas leer cualquier documentación y hacer tu primera petición.

Video — 19 min

Google Sheets API: leer y escribir desde Python

Conectas Python directamente con Google Sheets sin descargar nada. Lees un reporte, calculas métricas y escribes los resultados en una nueva pestaña. Todo automático.

WhatsApp Business API: enviar mensajes desde Python

Generar PDFs automáticos con Python
